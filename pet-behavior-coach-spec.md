# 构建规格 · AI 宠物行为训练教练(MVP Demo)

> 给 Codex/Cursor 的指令:严格按本文件实现。目标是**一个可演示、可录屏的 Web Demo**,不是生产系统。
> 凡标【DEMO 范围外】的一律不做,避免过度工程。总工时目标:1 天内跑通。

---

## 0. 一句话产品
用户描述(+可选照片/短视频)自家狗/猫的问题行为 → AI 输出**行为根因诊断 + 分级 + 4–8 周个性化训练计划 + 每日任务 + 安全红线**。区别于"看免费视频"(不个性化)和"请上门训犬师"(贵),AI 提供**个性化、随时可用、可追踪**的中间方案。

## 1. MVP 目标用户(Demo 只服务这一类,别扩)
养狗新手 / 上班族,面对高频行为问题:**拆家、分离焦虑、乱叫、随地大小便、扑人/咬人、不亲人**。猫可作为下拉选项存在但内容以狗为主。

## 2. 核心流程(一条线,别加分支)
```
输入表单  →  Stage1 结构化  →  Stage2 行为诊断  →  Stage3 训练计划生成  →  结果页渲染
```
- 输入:文字描述(必填)+ 宠物基本信息(犬种/年龄/性别是否绝育)+ 可选上传 1 张照片(多模态)
- 输出:结果页展示诊断 + 训练计划 + 每日任务卡 + "何时该找专业人士"红线提示
- 【DEMO 范围外】用户账号、数据库持久化、支付、真正的进度追踪回写。进度页用假数据静态展示即可。

## 3. AI 工作流(本项目的核心资产 —— 三段式 Prompt 链)

三次 LLM 调用串行。Stage1/2/3 都要求模型**只输出 JSON**(便于前端渲染)。

### Stage 1 — 意图结构化(把用户口语整理成结构化画像)
System:
```
你是宠物行为评估助手。用户会用口语描述宠物的问题行为,可能信息零散、混杂情绪。
你的任务:抽取并结构化为下方 JSON,不做任何建议,不臆造未提供的信息(缺失填 null)。
输出且仅输出 JSON。
```
User(模板):
```
宠物类型: {dog/cat}
品种: {breed}
年龄: {age}
是否绝育: {bool}
用户描述: "{free_text}"
（若有图片一并传入）
```
期望输出 JSON:
```json
{
  "species": "dog",
  "breed": "柯基",
  "age_months": 8,
  "neutered": false,
  "primary_behavior": "分离焦虑",
  "symptoms": ["独处时嚎叫", "抓门", "破坏家具"],
  "triggers": ["主人出门", "独处超过30分钟"],
  "already_tried": ["买了玩具但没用"],
  "severity_signals": ["自残倾向: 无", "持续时长: 2周"],
  "image_observations": "照片显示门框有抓咬痕迹"
}
```

### Stage 2 — 行为根因诊断(输入 Stage1 的 JSON)
System:
```
你是资深犬猫行为咨询师(非兽医)。基于结构化行为画像,给出最可能的行为学根因分析。
规则:
1. 给出 1 个主根因 + 至多 2 个次要因素,每个附一句通俗解释。
2. 严格区分"行为问题"与"可能的医疗问题"。若症状提示疾病(如突然乱尿可能泌尿感染、老年犬认知障碍等),必须在 medical_flag 标注并建议就医。
3. 输出严重度分级:low / medium / high。high = 涉及攻击伤人或自残,需线下专业介入。
4. 不臆造,信息不足就在 confidence 里说明。
只输出 JSON。
```
期望输出 JSON:
```json
{
  "root_cause": {"label":"分离焦虑","explain":"独处时的过度应激反应,而非报复或调皮"},
  "secondary_factors": [{"label":"精力未释放","explain":"柯基精力旺盛,运动量不足会放大焦虑"}],
  "severity": "medium",
  "medical_flag": {"is_flagged": false, "reason": null},
  "confidence": "中等,建议补充独处时的录像进一步确认"
}
```

### Stage 3 — 个性化训练计划(输入 Stage1+Stage2)
System:
```
你是宠物行为训练课程设计师。基于画像和诊断,生成一套可执行的分阶段训练计划。
要求:
1. 按周分阶段(4–8 周),每周一个目标,循序渐进。
2. 每周给 2–3 个"每日可做的小任务",描述具体到"做什么/多久/成功标准"。
3. 列出所需道具(尽量家里现成)。
4. 必须包含 "red_lines":出现哪些情况应立即停止自训并寻求持证训犬师/兽医(如攻击升级、自残、超过设定周期无改善)。
5. 语气鼓励、可执行,避免专业术语堆砌。
只输出 JSON。
```
期望输出 JSON(结构):
```json
{
  "plan_title": "柯基分离焦虑 6 周脱敏计划",
  "goal": "让狗狗独处时保持平静,逐步延长可独处时间",
  "weeks": [
    {"week":1,"theme":"建立安全区","daily_tasks":[
        {"task":"关笼/围栏内喂食,营造正向联想","duration":"每天2次,每次10分钟","success":"进入不抗拒即达标"}
      ],"tools":["零食","狗笼"]}
  ],
  "red_lines":["出现啃咬到出血等自残行为立即停止并就医","6周无任何改善建议转持证训犬师"],
  "disclaimer":"本计划为行为训练辅助,不替代兽医诊疗。"
}
```

> 说明:免责声明只在"医疗边界"出现一次,不要每屏都贴——那会削弱产品可信度。定位清楚是"训练教练"而非"看病"。

## 4. 技术栈(选最快能出好看 Demo 的)
- **首选:Next.js(App Router)+ 单个 API route + Tailwind**。一个页面 = 输入表单 + 结果展示;`/api/coach` 里串行调三次模型。前端好看、录屏专业、部署 Vercel 一键出可分享链接。
- 备选(想更快):**Streamlit / Gradio** 单文件,牺牲美观换速度。
- 模型:多模态用 `claude-opus-4-8` 或 `gpt-4o` 系列(照片理解);纯文本阶段可用更便宜的模型省钱。Key 走环境变量 `.env.local`,别硬编码。

## 5. 文件结构(Next.js 方案)
```
/app
  page.tsx              // 输入表单 + 结果页(单页,useState 切换)
  /api/coach/route.ts   // 串行编排 Stage1→2→3,返回合并 JSON
/lib
  prompts.ts            // 三段 system prompt(照抄本文件第3节)
  llm.ts                // 封装模型调用(多模态入参)
/components
  IntakeForm.tsx
  DiagnosisCard.tsx     // 渲染 Stage2
  TrainingPlan.tsx      // 渲染 Stage3(周计划手风琴 + 每日任务卡)
  RedLineAlert.tsx      // 红线警示条
.env.local              // API keys
```

## 6. 端到端样例(Codex 用它自测)
输入:「我家柯基 8 个月,没绝育,我一出门它就在家嚎叫抓门,家具都咬坏了,买了玩具也没用,大概两周了」+ 一张门框咬痕照片
期望:结果页显示 → 根因"分离焦虑(中度)" + 6 周脱敏计划(每周目标+每日任务) + 红线提示 + 一次性免责声明。全程 < 15 秒返回。

## 7. 构建顺序(先跑通再美化)
1. 搭 Next.js 骨架 + Tailwind,放好 `prompts.ts`。
2. `llm.ts` 打通单次模型调用(先纯文本)。
3. `/api/coach` 串起三段,用样例输入在终端验证 JSON 正确。
4. 前端表单 → 调 API → 渲染结果卡片。
5. 加多模态(照片)。
6. 美化 + 加载动画 + 录屏。
7. 【DEMO 范围外,时间够再说】进度追踪静态页、猫的内容。
